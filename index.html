Introduction

0WM, pronounced /zʋʊm/ or /oʊm/, is a next-generation Wi-Fi mapping solution using off-the-shelf components and open standards to perform hassle-free, large-scale Wi-Fi surveys.

0WM is currently in its pre-alpha stage. Please check back from time to time to see our progress.

Wi-Fi has become so popular in recent years that your devices, at home or at work, are less likely to be connected to the Internet using a wire. Laptops themselves are slowly losing their 8P8C (RJ45 / Ethernet) ports. And yet, deploying a reliable Wi-Fi network is no easy task, because of walls, urban wave pollution, or even weather radars. Professional Wi-Fi survey tools are so expensive that private individuals, associations and small businesses cannot afford them, and free ones are too cumbersome.

0WM makes high-quality Wi-Fi surveys affordable to everyone. Why would you need a dedicated device for Wi-Fi surveys when you already have a phone packed with sensors? Why would you need a dedicated wireless sensor when you already have factory-calibrated omnidirectional APs?

Getting started

0WM is not ready for use by non-developers. The following instructions will help you set up a development environment to tinker with it. We present here the following components:

Prerequisites / What You Need

  • A machine capable of running an OCaml server (requires dune and opam)
  • A smartphone with an AR-capable browser (rooted Android or iOS with XRViewer)
  • An OpenWRT-compatible Access Point (or the Mock AP)
  • A USB-C to Ethernet dongle to wire the phone to the AP

0WM Server

The central backend of 0WM handles the core logic. It receives scan data, stores project and session information, and exposes REST and WebSocket APIs to serve heatmaps and floorplans to the frontends.

For full installation and configuration instructions, please refer to the 0WM Server repository.

Note on SSIDs: By default, the ssids list in the server configuration is empty. As a result, any heatmaps generated during testing will be empty (they will just appear blue). Make sure to add the SSIDs you want to survey into the configuration to generate meaningful heatmaps.

Note that apart from floorplan images, the server’s internal storage is not persistent yet; hence, stopping the server leads to data loss (remember, this is a development environment!).

0WM OpMode

This is an operator dashboard frontend that communicates with the server to upload floorplans, edit boundaries, and position them on a map with precise georeferencing.

When you add a new project, you'll be prompted to upload a file. Select a clear, top-down floorplan image (PNG, JPEG, WebP) of the building and draw boundaries and walls. In the Map Editor, you will place 3 anchor points to map the floorplan to the real world. We recommend placing these points in the corners of the building to ensure accurate linear transformation without distortion. Lastly, provide the altitude parameters (you only need to fill in 2 of the 3 fields: altitude, ceiling, or height).

For full installation and deployment instructions, please refer to the 0WM OpMode repository.

The OpMode can be configured with the config.json file at the root of the repository.

0WM Client

This is a mobile survey frontend that allows you to perform real-time Wi-Fi surveys. It fetches floorplans from the server, retrieves Wi-Fi scan data from the AP (Access Point), uses WebXR to track movement, and streams measurements back to the server in real-time.

For full installation and deployment instructions, please refer to the 0WM Client repository.

The client can be configured with the config.json file at the root of the repository.

0WM AP

This provides the software and configuration to properly set up an AP (Access Point) endpoint to expose Wi-Fi scans for 0WM. You can use the 0WM Mock AP for testing purposes.

For detailed instructions on configuring both mock and real OpenWRT Access Points, please see our dedicated AP Configuration Guide.

Survey Workflow

Once your components are running in your development environment, a typical survey workflow operates as follows:

  1. Start the 0WM Server to act as the central hub.
  2. In 0WM OpMode, prepare your setup: upload a floorplan image, draw walls and boundaries, define your geo-anchors and altitude parameters, and synchronize this metadata to the server.
  3. In the 0WM Client, connect to both the server and the AP. Select your mapped floorplan, initialize WebXR tracking by scanning your room, and walk the surveying area to capture and stream Wi-Fi measurements back to the server.

Running the client

Testing the 0WM client requires specific network and browser configurations depending on whether you are running it on a mobile device (Android vs. iOS) or on a computer using a WebXR emulator.

Please see the detailed Running the Client Guide for complete instructions and configuration examples.
